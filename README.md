### Структура проекта

```bash
first_project_template/
│
├── first_project_template/         # Основной код проекта
│   ├── artifacts/
│   │   ├── best_params.json        # Обученные на тренировочных данных параметры стратегии, создается в src/strategy
│   │   ├── backtest_metrics.json   # Бэктестовые метрики стратегии, создается в src/run_backtest
│   │   └── equity_curve.html       # График PnL стратегии в формате HTML, создается в src/run_backtest
│   ├── data/
│   │   ├── train_data.csv      # Котировки обучающей выборки, создаются в src/get_data
│   │   └── backtest_data.csv   # Котировки бэктеста, создаются в src/get_data
│   └── src/                    # Исходный код
│       ├── get_data.py         # Здесь скачиваются данные
│       ├── run_backtest.py     # Здесь происходит инференс твоей стратегии
│       ├── strategy.py         # Здесь задаются правила, по которым будет торговать твоя стратегия
│       └── utils.py            # Вспомогательные модули
├── main.py                     # Основной файл проекта, запускающий цикл стратегии от обучения до оценки метрик
├── config.yaml                 # Конфиг с требованиями к бэктесту
├── pyproject.toml              # Конфиг проекта, нужен нам для Poetry
│
└── README.md                   # Файл, который ты сейчас читаешь
```


## Установка и запуск

### Склонируйте себе репозиторий

Выберите способ работы с репозиторием

#### Вариант 1 - Используя Git:
-  Установите Git: git-scm.com
- Склонируйте проект:

 Скопируйте URL вашего репозитория:

- Откройте ваш репозиторий на GitHub
- Нажмите зеленую кнопку "Code"
- Скопируйте ссылку (HTTPS)

В терминале введите:
```bash
git clone https://github.com/school-of-quants/mid-frequency-trading-project-1-{YOUR_USERNAME}.git
```

От вас потребуется username и password. Password здесь это не пароль от вашего аккаунта на гитхабе, а токен, который вам нужно сгенерировать в настройках аккаунта. 

[Инструкция](https://www.geeksforgeeks.org/git/how-to-generate-personal-access-token-in-github/) как это сделать 

1.3 После того как поработаете над проектом, вы можете сделать коммит из терминала в свой репозиторий на гитхабе. 
Чтобы сделать коммит, перейдите в свой склонированный репозиторий

```bash
cd mid-frequency-trading-project-1-{YOUR_USERNAME}
```

Затем вам нужно подготовить изменения с помощью git add, зафиксировать их локально командой git commit -m "Ваше сообщение" и затем отправить на GitHub с помощью git push

#### Вариант 2

Если не получилось разобраться с git, вы можете скачать репозиторий как архив: нажмите зеленую кнопку "Code" и выберите Download ZIP. 

После того как закончите работу над проектом, выберите Add files и загрузите обновленные файлы в репозиторий.


!!! Обязательно Обязательно перепроверьте перед дедлайном, что все изменения в файлах отражены в вашем репозитории. Ничего дополнительно сдавать не нужно, после дедлайна возможность коммитить в репозиторий закроется, преподаватель будет видеть вашу работу

### Требования

Надеюсь, у тебя уже установлен Python. Если нет, то сделай это по [инструкции](https://practicum.yandex.ru/blog/kak-ustanovit-python-na-kompyuter/)

Для запуска проекта необходимо установить зависимости. Для этого будем использовать Poetry

```bash
pip install poetry==1.8.5
```

Следом ставим зависимости проекта
```bash
poetry install
```

## Что нужно сделать

Полные требования к проекту указаны на предпоследнем слайде первой лекции нашего курса.

Вы можете модифицировать данный шаблон в случае необходимости, однако верхнеуровнево вам необходимо:

0. Внимательно ознакомиться с кодом проекта.
1. Прописать скрипт для получения тренировочных и бэктестовых даннных (`src/get_data.py`). В случае, если метод получения ваших данных не подразумевает использования API (например вы скачиваете их вручную с сайта), обязательно укажите это в DESCRIPTION.md в секции 0, а также укажите ссылку на источник датасета.
2. Определить класс стратегии в `src/strategy.py`. 
    - Переименовать класс
    - Переопределить методы `init` и `next` соответствующим вашей стретегии образом. Если вы собираетесь обучать параметры на обучающей выборке, требуется также переопределить метод `find_best_params`
    - Добавить в `src/run_backtest.py` расчет беты вашей стратегии, обосновать выбор рыночного портфеля в секции 4 DESCRIPTION.md 
    - Запустить main.py и проанализировать метрики вашей стратегии
3. По итогу проделанной работы заполнить файл DESCRIPTION.md
4. Закоммитить изменения в вашу локальную копию репозитория.
